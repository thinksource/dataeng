Assignment

Given: source data file (members-data.csv),

Build an ETL pipeline with a Test-Driven Development (TDD) approach by performing the following tasks:

  • Read data from a file(s) having pipe separated data (data)

  • Transform data to data model

  • Load the data into any database or store as document collection to disk

Requirements

  • Python 3.8+
  • MongoDB / json documents

minimal third party libraries

Instructions

Reading Data from the given file

Create a read_data() method that takes a file path as an argument and reads the file. Return an in-memory data structure such as a list of dictionaries or a Pandas DataFrame.

column names types
FirstName string
LastName string
Company string
BirthDate number
Salary number
Address string
Suburb string
State string
Post number
Phone number
Mobile number
Email string

Transforming data

Create a transform_data() method that takes the in-memory data structure from the previous step as an argument. Perform the following transformation tasks:

  • Convert the BirthDate from the format YYYY-MM-DD to DD/MM/YYYY, Salary in $ format with commas

  • Do some cleaning on FirstName and LastName columns when needed and remove any leading/trailing spaces.

  • Merge the FirstName and LastName columns into a new column named FullName.

  • Calculate each employees's age from the BirthDate column using as reference Mar 1st, 2024. Add a new column named Age to store the computed age.

  • Add a new column named SalaryBucket to categorize the employees based on their salary as follows:

    • A for employees earning below 50.000
    • B for employees earning between 50.000 and 100.000
    • C for employees earning above 100.000

  • Drop columns FirstName, LastName

    build nested entity class to have address as nested class.

Loading the Data into MongoDB or save as json file

Create a load_data() method that takes the transformed data as an argument, connects to the MongoDB database and inserts the given data into a predefined collection.

Docker Compose [Optional]

Create a Dockerfile to create a Docker image of your Python application.

Set up a docker-compose.yml file where you will define the services. Configure two services:

  • Python application with your ETL pipeline and
  • MongoDB database.

Ensure that both services can communicate with each other.

Deliverables

The minimum we expect to see in your submission is:

  • main.py, the main ETL script containing the core methods: read_csv(), transform_data(), and load_data().
  • Test suites for everything you think should be tested.
  • package dependencies
  • A Dockerfile to create a Docker image of the Python application [Optional].
  • A docker-compose.yml file defining the required services [Optional].
  • A README.md file with instructions for setting up and running the ETL pipeline using Docker Compose.

Submission

Please submit your assignment as a link to a GitHub repository containing the required files. You can also create a private repository and share it with us.

Notes

Feel free to use any framework, pattern to implement the ETL pipeline and document your approach accordingly in the README.md file.
